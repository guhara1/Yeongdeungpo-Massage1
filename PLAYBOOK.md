# 지역 출장마사지 사이트 제작 플레이북 (완전판)

> 노원 블랙 마사지(nowon-massage) 사이트의 메뉴·페이지 작성법·디자인·SEO 규칙 전체.
> 이 문서 하나로 다른 지역(예: 도봉, 강북, 의정부)에도 동일한 사이트를 복제할 수 있다.

---

## 0. 핵심 원칙 (절대 규칙)

```
상단 메뉴는 짧게 — 키워드 반복 금지, 지역명·역명만 표시
지역은 대표 동 단위만 — 숫자 행정동 페이지 금지 (월계1동 ❌ → 월계동 ⭕)
지하철역은 역 1개당 페이지 1개 — 환승역도 URL 하나, 출구별 페이지 금지
지역+역+테마 조합 페이지 금지 (노원역 스웨디시 ❌)
모든 색인 페이지 본문 2,000~2,500자 (공통 요금 블록 제외 측정)
2,000자 미만은 자동 noindex (빌드가 강제)
지역명만 바꾼 복붙 페이지 금지 — 페이지 간 유사도 5% 이하 유지
푸터에 지역명·역명 대량 나열 금지
타이틀은 "지역명 출장마사지·홈타이 | …" 키워드 선두 배치
```

## 1. 기술 구조

```
build.py            # 빌드 스크립트 — 레이아웃·TOC 자동생성·글자수 검사·sitemap
content/
  site.py           # BASE_URL·상호·전화·메뉴(NAV) 정의
  main.py           # 메인 페이지 (+ LocalBusiness/FAQPage JSON-LD, 히어로)
  areas.py          # 지역: 허브 1 + 대표 동 N
  stations.py       # 역: 허브 1 + 역 N
  themes.py         # 테마: 허브 1 + 테마 14
  info.py           # 출장마사지 안내·코스·예약·가이드·후기·고객센터·약관
  magazine.py       # 매거진 허브 + 아티클 (Article JSON-LD)
  about.py          # 운영자 소개 (E-E-A-T)
  pricing.py        # 공용 요금 블록 (21페이지 삽입)
assets/
  style.css         # 전체 디자인 시스템
  nav.js            # 모바일 메뉴 + TOC 스크롤스파이
  favicon.svg/ico/png, og-image.png, apple-touch-icon.png
```

빌드: `python3 build.py` → 페이지별 글자수 리포트 출력.
페이지 정의는 dict 하나:

```python
{
  "path": "nowon-gu/wolgye-dong/",   # 끝에 / (메인은 "")
  "title": "...",                    # SEO 타이틀
  "desc": "...",                     # 메타 디스크립션 (50~160자)
  "h1": "...",
  "body": "...HTML...",              # <section><h2>…</h2><p>…</p></section> 반복
  "breadcrumb": [("지역별 안내", "/nowon-gu/"), ("월계동", None)],
  "extra_head": "...",               # JSON-LD 등 (선택)
  "hero": "...",                     # 히어로 (메인 전용, 선택)
  "noindex": True,                   # 강제 noindex (약관 등, 선택)
}
```

빌드가 자동으로 해주는 것: H2 섹션 id 부여 → 좌측 목차(TOC) 생성, 섹션 골드 넘버링(01, 02…),
2,000자 미만 noindex + sitemap 제외, 2,500자 초과 경고, canonical/og:image/파비콘 head 삽입.

## 2. 메뉴 구성 (상단 메뉴 + 하위메뉴 전체)

하위 메뉴에는 "○○동 출장마사지"처럼 키워드를 반복하지 않고 **지역명·역명만** 쓴다.

```
홈                /
노원 출장마사지     /massage/          ← 하위는 모두 앵커(#)
 ├ 출장마사지 안내   /massage/#service
 ├ 홈타이 안내      /massage/#hometai
 ├ 전지역 방문 안내  /massage/#coverage
 ├ 지하철역 인근 안내 /massage/#stations
 ├ 예약 가능 시간    /massage/#hours
 ├ 코스 선택 안내    /massage/#course
 ├ 이용 전 확인사항  /massage/#check
 ├ 위생·안전 안내    /massage/#safety
 └ 자주 묻는 질문    /massage/#faq
지역별 안내        /nowon-gu/
 ├ 노원구 전체 / 월계동 / 공릉동 / 하계동 / 중계동 / 상계동
지하철역별 안내     /nowon-gu/stations/
 ├ 역 전체 + 13개 역 (노원역·상계역·불암산역·수락산역·마들역·중계역·하계역·
 │  공릉역·태릉입구역·석계역·광운대역·월계역·화랑대역)
테마별 안내        /themes/
 ├ 전체 테마 + 14개 (스웨디시/로미로미/타이마사지/중국마사지/아로마테라피/홈케어/
 │  호텔식마사지/발마사지/스포츠·경락/스킨케어/왁싱/커플 관리/24시간/수면 가능)
코스안내          /courses/           ← 하위는 앵커
 ├ 전체 코스/피로 회복/아로마/스포츠/홈타이/커플·가족/기업·단체/가격/선택 가이드
예약안내          /reservation/       ← 앵커: how/hours/place/payment/change/check
이용가이드        /guide/             ← 앵커: first/prepare/hygiene/after/prohibited/faq
매거진            /magazine/          ← 하위: 전체 글 + 각 아티클
후기              /reviews/           ← 앵커: area/station/write
고객센터          /support/           ← 앵커 + privacy/terms 별도 페이지
```

**원리**: 예약·가이드·후기·고객센터의 하위 항목은 별도 페이지로 만들면 얇은 페이지가
되므로 **한 페이지 + 앵커 섹션**으로 운영한다. 약관 2종만 별도 페이지(noindex).

## 3. 디자인 시스템 — "다크 럭스 스파"

> 다크 네이비 배경 + 골드 강조를 본문까지 확장. 좌측 고정 목차(TOC), 높은 대비.
> 대담하고 고급 호텔 스파 같은 무드.

### 3-1. 컬러 토큰 (CSS 변수)

```css
:root {
  --bg: #070b14;          /* 가장 깊은 네이비 블랙 (body) */
  --bg-2: #0a1120;        /* 헤더·푸터 */
  --panel: #0d1626;       /* 본문 섹션 패널 */
  --panel-2: #101b30;     /* 카드 */
  --line: #1b2742;        /* 네이비 라인 */
  --line-gold: rgba(200,162,94,.28);
  --gold: #c8a25e;  --gold-soft: #e9d7ab;  --gold-deep: #a9853f;
  --text: #eaedf4;  --text-dim: #97a1b8;
  --serif: "Noto Serif KR", serif;   /* 모든 헤딩·브랜드·숫자 */
  --sans: "Noto Sans KR", sans-serif; /* 본문 */
  --max: 1180px;
}
```

폰트는 Google Fonts: `Noto+Sans+KR:wght@400;500;700` + `Noto+Serif+KR:wght@600;700;900`.
버튼 텍스트 컬러(골드 위)는 `#120e06`.

### 3-2. 헤더 (2단 구조)

```html
<header class="site-header">
  <div class="header-accent"></div>            <!-- 흐르는 골드 라인 3px -->
  <div class="header-top">
    <div class="header-inner">
      <a class="brand"><span class="brand-mark">N</span>
        <span class="brand-text">노원 블랙 마사지</span></a>     <!-- 골드 메탈 그라데이션 텍스트 -->
      <p class="header-tagline">◆ 노원구 전지역 방문 관리 ◆ 24시간 상담</p>
      <a class="header-call" href="tel:...">예약전화 | 0508-202-4719</a>  <!-- 골드 알약 -->
      <button class="nav-toggle"><span></span><span></span><span></span></button>
    </div>
  </div>
  <nav class="main-nav"><ul class="nav-list">…메뉴…</ul></nav>  <!-- 전용 행, 중앙 정렬 -->
</header>
```

디자인 포인트:
- **배경**: 미드나잇 블루 대각 그라데이션 `linear-gradient(115deg,#0c1530,#15244c 34%,#101d3e 58%,#0a1228)`
  위에 좌측 골드 글로우 + 우측 로열블루 글로우(라디얼) 2~3겹.
- **header-accent**: `linear-gradient(90deg, transparent, #6e5526, gold, #f4e8c6, gold, #6e5526, transparent)`
  `background-size:200%` + 7초 background-position 애니메이션 = 빛이 흐르는 효과.
- 브랜드 마크: 골드 그라데이션 채움 원 + 이중 글로우 box-shadow.
- 메뉴 hover: 가운데서 펼쳐지는 골드 언더라인 (`::after scaleX(0→1)`).
- 드롭다운: `--bg-2` 배경, 골드 보더, hover 시 표시.
- 모바일(≤920px): 햄버거(3선→X), 메뉴 세로 아코디언, 하위 메뉴 ▾.

### 3-3. 푸터 (구글 신뢰 신호형 4컬럼)

```html
<footer class="site-footer">
  <div class="container footer-grid">          <!-- minmax(260px,1.4fr) 1fr 1fr 1fr -->
    <div class="footer-col footer-about">
      상호 / 한 줄 소개 / <address>예약전화(tel:)·상담시간·서비스 지역</address>
    </div>
    <nav aria-label="서비스 안내">서비스: 출장마사지/지역별/역별/테마별/코스 (허브 5개만)</nav>
    <nav aria-label="이용 안내">예약안내/이용가이드/후기/고객센터/FAQ</nav>
    <nav aria-label="정책 및 기준">운영자 소개/개인정보처리방침/이용약관/위생 기준/금지행위/제휴</nav>
  </div>
  <div class="footer-bottom">© 상호 │ 운영 원칙 고지 │ 웹사이트 제작문의 ↗(외부, nofollow)</div>
</footer>
```

금지: 푸터에 동·역 이름 대량 나열, "○○동 출장마사지" 앵커텍스트.

### 3-4. 본문 레이아웃 (좌측 고정 목차)

- 그리드: `216px + 1fr`, gap 48px. 1080px 이하에서 TOC 숨김.
- TOC: `position:sticky; top:132px`, 항목은 H2에서 자동 생성(빌드),
  스크롤스파이(IntersectionObserver)로 현재 섹션 골드 하이라이트.
- **모든 섹션 = 네이비 패널 카드**: `--panel` 그라데이션, 보더 `--line`, radius 16px,
  padding 34px 38px. H2 위에 골드 세리프 번호(01, 02…)가 CSS counter로 자동 표시.
- H1: 세리프 2.05rem + 하단 64px 골드 바. H2: 세리프 1.42rem + 40px 골드 밑줄.
- FAQ: `.faq-item` — Q./A. 접두가 CSS ::before 골드로 자동.
- CTA 섹션(`.cta`): 골드 글로우 패널 + 골드 그라데이션 전화 버튼(알약 34px radius).
- 카드 그리드(`.card-grid`): auto-fill minmax(148px,1fr), hover 시 골드 보더+부상.

### 3-5. 히어로 (메인 전용)

```html
<section class="hero">
  <p class="hero-badge">PREMIUM VISITING SPA · 노원구 전지역</p>   <!-- 자간 0.34em -->
  <h1>노원 출장마사지·홈타이<br>예약 안내</h1>                      <!-- 세리프 3rem -->
  <p class="hero-lead">두 줄 카피</p>
  <div class="hero-actions">📞 전화(골드) + 코스 보기(아웃라인)</div>
  <ul class="hero-stats">5개 대표 지역 │ 13개 역세권 │ 14개 테마 │ 24시간 상담</ul>
</section>
```

배경: 상단 중앙 골드 라디얼 글로우 + 하단 네이비, 상단 중앙에 수직 골드 라인 58px.

### 3-6. 요금 컴포넌트 (content/pricing.py — 21페이지 공용)

```
H2: 코스별 기본 요금
리드: "60·90·120분 코스별 기본 요금입니다. 숨겨진 추가 비용 없이 투명하게 안내합니다."
카드 3개 (price-grid):
  60분  90,000원   기본 컨디션·릴랙스 케어     [예약 문의-아웃라인]
  90분 150,000원   아로마 포함 추천 구성       [추천배지][예약 문의-골드]  ← featured
 120분 180,000원   전신 집중 프리미엄 케어     [예약 문의-아웃라인]
고지: "지역·예약 시간대·이동 거리에 따라 상담 시 최종 확인됩니다. 상세 요금 안내 보기 →"
```

배치: 메인 + 지역 허브/동 + 역 허브/역 = 21페이지, 항상 **CTA 바로 위**.
모바일 1열 전환 시 featured 카드가 맨 위(order:-1).
숫자 변경 시 pricing.py 한 곳만 수정. `/courses/#price` 표와 LocalBusiness
JSON-LD `priceRange`도 함께 맞출 것. **요금 블록은 글자수 측정에서 제외됨.**

### 3-7. 우측 하단 전화 FAB

64px 골드 원형(모바일 56px), 수화기 SVG가 벨처럼 흔들리는 애니메이션(2.4s) +
골드 링 2겹 확산(`::before/::after` scale 1→1.85 페이드). hover 시 좌측 "예약 전화" 라벨.
`prefers-reduced-motion` 대응 필수.

### 3-8. 파비콘·OG 이미지

- 파비콘: 네이비 원 + 골드 링 + 세리프 N (SVG + ICO + PNG 16/32/180/192/512).
- OG: 1200×630, 동일 마크 + 라틴 브랜드명 + 전화. 전 페이지 `og:image` +
  `twitter:card=summary_large_image` + JSON-LD `image` 로 선호 썸네일 지정.

## 4. 메인 페이지 작성법

```
URL: /        H1: 노원 출장마사지·홈타이 예약 안내
SEO Title: 노원 출장마사지·홈타이 | 노원구 전지역 방문 마사지 예약 안내
Description: 노원 출장마사지·홈타이 안내 페이지입니다. 월계동, 공릉동, 하계동,
  중계동, 상계동과 노원구 주요 지하철역 인근, 테마별 관리, 예약 전 확인사항을 확인해보세요.
Focus: 노원 출장마사지 / 보조: 노원 홈타이, 노원구 방문 마사지, 역·동 키워드
```

섹션 구성과 글자수 배분 (총 2,100~2,500자):

| # | H2 | 분량 | 작성 방향 |
|---|----|------|----------|
| 1 | 노원 출장마사지·홈타이 서비스 안내 | 250~300 | 허브 역할 선언, 상세는 하위 페이지로 |
| 2 | 노원구 전지역 방문 가능 안내 | 250~300 | 대표 동 5개 + 숫자 행정동 통합 이유 |
| 3 | 지역별 안내 | 250~300 | 카드 5개(동 이름만!) + 설명 |
| 4 | 지하철역 인근 안내 | 250~300 | 카드 13개(역 이름만) + 단일 페이지 원칙 |
| 5 | 테마별 관리 안내 | 250~300 | 카드 14개 + 조합 페이지 안 만드는 이유 |
| 6 | 코스 선택 안내 | 200~250 | 목적·컨디션 기준 제시, 코스안내로 연결 |
| 7 | 예약 진행 방식 | 200~250 | 5단계: 위치→시간→코스·인원→가능 여부→확정 |
| 8 | 이용 전 확인사항 | 200~250 | 주소·공동현관·주차·조용한 공간 |
| 9 | 위생 및 안전 안내 | 200~250 | 위생 기준·개인정보·불법 요청 거부 명시 |
